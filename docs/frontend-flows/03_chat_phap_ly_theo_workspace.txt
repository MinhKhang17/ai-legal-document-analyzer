FLOW 03 - CHAT PHÁP LÝ THEO WORKSPACE

Mục tiêu:
- Cho customer hỏi đáp pháp lý trên workspace, gắn với document cụ thể hoặc toàn workspace, xem lịch sử session và có thể escalated sang legal ticket.

Actor:
- `CUSTOMER`

Điểm bắt đầu:
- Route `/chat`

Điều kiện tiên quyết:
- Đã có workspace
- Với workspace thường: backend yêu cầu phải có ít nhất một document `READY` nếu không chọn document cụ thể

Luồng chính:
1. `LegalChatPage` load workspaces và loại workspace hệ thống khỏi list hiển thị.
2. FE load song song documents và chat sessions của workspace đang chọn.
3. FE chọn:
4. `selectedDocumentId` = document từ query param hoặc document `READY` đầu tiên
5. `selectedSessionId` = session từ query param hoặc default session hoặc session đầu tiên
6. FE load lịch sử message theo `chatSessionId`.
7. Người dùng nhập câu hỏi và submit.
8. Nếu đã có `chatSessionId`, FE gọi `sendChatSessionMessage()`.
9. Nếu chưa có session, FE có thể dùng `sendWorkspaceMessage()` để backend tự tạo default session.
10. Backend lưu user message, gọi Python AI client, lưu assistant message.
11. FE render câu trả lời, metadata, nút `Chi tiết`, citation và action tạo legal ticket nếu assistant đề xuất.
12. Người dùng có thể:
    - tạo session mới
    - đổi tên session
    - xóa session
    - xem summary/memory của session
    - mở chi tiết message
    - tạo legal ticket từ assistant message
    - **Đánh giá câu trả lời LLM:** Dưới mỗi tin nhắn của AI Assistant, người dùng có thể bấm Thích/Không thích (hoặc chấm điểm từ 1-5) và gửi bình luận. FE gọi API gửi phản hồi.
    - **Chia sẻ cuộc hội thoại:** Tại góc màn hình chat, khách hàng có thể nhấn nút "Chia sẻ". FE gọi API share, lấy về đường dẫn `/shared/chat/{shareToken}` và hiển thị cho khách hàng copy.

13. Xem cuộc hội thoại được chia sẻ (Dành cho Admin và Expert):
    - Admin hoặc Expert truy cập vào link `/shared/chat/{shareToken}`.
    - Hệ thống kiểm tra quyền đăng nhập của Admin/Expert. Nếu đúng vai trò, hiển thị lịch sử cuộc hội thoại dưới dạng Read-Only.
    - Giao diện này **không chứa khung nhập tin nhắn (prompt)** và không cho phép chỉnh sửa/xóa phiên chat.

Sơ đồ text:
- `select workspace/document/session -> load messages -> ask question -> AI response -> optional create legal ticket`
- `AI Response -> Click Rate Star/Thumb -> Send Feedback API -> Save Feedback`
- `Chat Screen -> Click Share -> Call Share API -> Copy URL -> Send to Admin/Expert`
- `Admin/Expert -> Go to Shared URL -> Verify auth -> Load Shared Chat History -> Render Read-Only UI (No Prompt)`

Màn hình / component:
- `frontend/src/pages/chat/LegalChatPage.tsx`
- `frontend/src/components/chat/ChatMessageContent.tsx`
- Component phản hồi/đánh giá tin nhắn LLM (ThumbsUp/Down hoặc Rating Stars dưới mỗi bubble chat) [MỚI]
- `frontend/src/pages/chat/SharedChatPage.tsx` (Màn hình xem hội thoại chia sẻ ở chế độ Read-Only dành riêng cho Admin/Expert) [MỚI]

Dữ liệu / state:
- `workspaces`, `workspaceDocuments`, `chatSessions`
- `selectedWorkspaceId`, `selectedSessionId`, `selectedDocumentId`
- `messages`, `input`, `sending`
- `shareUrl`, `sharingSessionId`
- `sharedChatMessages`, `sharedSessionDetail` (cho SharedChatPage)

Validation:
- Message không được rỗng
- Backend `ChatMessageServiceImpl.validateMessageRequest()` giới hạn message <= 5000 ký tự
- `documentId` nếu có phải thuộc workspace và phải ở trạng thái `READY`
- Chỉ hiển thị nút Chia sẻ với khách hàng sở hữu phiên chat.

API / service tích hợp:
- FE chat: `frontend/src/services/chat.service.ts`
- FE AI citations: `frontend/src/api/aiFeatureApi.ts`
- FE legal ticket bridge: `frontend/src/services/legalTicket.service.ts`
- FE feedback: `frontend/src/services/feedback.service.ts`
- BE:
  - `POST /api/v1/workspaces/{workspaceId}/chat-sessions`
  - `GET /api/v1/workspaces/{workspaceId}/chat-sessions`
  - `GET /api/v1/chat-sessions/{chatSessionId}`
  - `PUT /api/v1/chat-sessions/{chatSessionId}`
  - `DELETE /api/v1/chat-sessions/{chatSessionId}`
  - `POST /api/v1/workspaces/{workspaceId}/messages`
  - `POST /api/v1/chat-sessions/{chatSessionId}/messages`
  - `GET /api/v1/chat-sessions/{chatSessionId}/messages`
  - `GET /api/v1/chat-messages/{messageId}`
  - `POST /api/v1/chat-messages/{messageId}/feedback` (Gửi phản hồi rating/comment) [MỚI]
  - `GET /api/v1/admin/chat-messages/feedback` (Admin xem toàn bộ đánh giá) [MỚI]
  - `POST /api/v1/chat-sessions/{chatSessionId}/share` (Tạo mã chia sẻ hội thoại) [MỚI]
  - `GET /api/v1/shared/chat/{shareToken}` (Lấy tin nhắn của hội thoại chia sẻ) [MỚI]

UI states:
- Không có workspace: nhắc chọn workspace
- Không có message: empty state
- AI fail: render assistant bubble lỗi và có nút retry
- Giao diện Shared Chat: Chỉ hiện danh sách tin nhắn, không có input text để chat.

Edge cases:
- Link chia sẻ bị truy cập bởi người không có quyền (Khách hàng khác hoặc người chưa đăng nhập): Trả về 403 Forbidden hoặc chuyển hướng về trang báo lỗi.
- Đánh giá tin nhắn nhiều lần: Ghi đè đánh giá trước đó hoặc trả về lỗi.

Business rules:
- Chat session đã chia sẻ chỉ cho phép Admin/Expert truy cập.
- Không thể chat thêm trong trang chia sẻ.

Quyền:
- Gửi tin nhắn và chia sẻ: `@PreAuthorize("hasRole('CUSTOMER')")`
- Gửi feedback: `@PreAuthorize("isAuthenticated()")`
- Xem danh sách feedback của User: `@PreAuthorize("hasRole('ADMIN')")`
- Xem shared chat: `@PreAuthorize("hasAnyRole('ADMIN', 'EXPERT')")`

FLOW 05 - ADMIN DUYỆT VÀ PHÂN CÔNG TICKET

Mục tiêu:
- Cho admin xem toàn bộ legal ticket, lọc theo status/risk, xem AI summary/chat/files và phân công hoặc từ chối ticket.

Actor:
- `ADMIN`

Điểm bắt đầu:
- Route `/admin/tickets`
- Route `/admin/tickets/:ticketId`

Điều kiện tiên quyết:
- User role admin

Luồng chính:
1. `AdminTicketsPage` tải danh sách ticket với filter status và risk.
2. FE gọi `getAdminLegalTickets(page, size, status?, riskLevel?)`.
3. Admin chọn một ticket để mở detail.
4. `AdminTicketDetailPage` tải song song:
   - ticket detail
   - admin AI summary
   - chat history
   - user files
   - AI assessment
   - AI summary/citations
   - Danh sách Expert từ API `getAdminExperts()`.
5. Admin chọn expert từ danh sách (hiển thị rõ Chuyên ngành - specialty và Ngành luật - legalDomain của từng Expert).
6. Admin gọi assign hoặc reassign.
7. Admin có thể nhập reject reason và từ chối ticket.

Sơ đồ text:
- `admin list -> filter -> open detail -> inspect evidence -> assign/reassign/reject`
- `load assign panel -> Call GET /admin/users/experts -> Render experts list with specialties/legal domain -> Select expert -> Call Assign API`

Màn hình / component:
- `frontend/src/pages/admin/AdminTicketsPage.tsx`
- `frontend/src/pages/admin/AdminTicketDetailPage.tsx` (Hiển thị chuyên ngành của Expert trong danh sách lựa chọn phân công)

Dữ liệu / state:
- `tickets`, `statusFilter`, `riskLevel`
- detail: `ticket`, `ticketSummary`, `chatMessages`, `files`, `assessment`, `aiSummary`, `citations`
- assignment: `experts` (chứa specialty, legalDomain), `selectedLawyerId`, `adminNote`
- rejection: `rejectReason`

Validation:
- Assign/reassign phải có `selectedLawyerId`
- Reject phải có `reason`
- Danh sách expert lấy trực tiếp từ API chuyên biệt `GET /api/v1/admin/users/experts` đã lọc sẵn expert hoạt động trên backend.

API / service tích hợp:
- FE:
  - `frontend/src/services/legalTicket.service.ts`
  - `frontend/src/services/user.service.ts`
- BE:
  - `GET /api/v1/admin/tickets`
  - `GET /api/v1/admin/tickets/{id}`
  - `GET /api/v1/admin/tickets/{id}/summary`
  - `GET /api/v1/admin/tickets/{id}/chat-history`
  - `GET /api/v1/admin/tickets/{id}/files`
  - `GET /api/v1/admin/users/experts` (Lấy danh sách expert kèm chuyên ngành) [MỚI]
  - `POST /api/v1/admin/tickets/{id}/assign-lawyer`
  - `POST /api/v1/admin/tickets/{id}/reassign-lawyer`
  - `POST /api/v1/admin/tickets/{id}/reject`

UI states:
- Dashboard thống kê tổng ticket/open ticket/current filter
- Detail có block AI summary, AI assessment, chat history, file list, citations
- Dropdown chọn expert: mỗi hàng hiển thị "Tên Expert (Chuyên ngành - Lĩnh vực)" để admin dễ dàng đối chiếu.

Edge cases:
- Nếu `getAdminExperts()` trả danh sách trống: hiển thị thông báo yêu cầu tạo tài khoản Expert trước.
- `hasAssignedLawyer` ảnh hưởng nút assign/reassign.

Business rules:
- Admin nhìn thấy toàn hệ thống, không chỉ ticket của mình.
- Phân công dựa trên sự tương đồng giữa ngành luật của Ticket và chuyên ngành của Expert.

Quyền:
- Tất cả endpoint trong flow này đều `hasRole('ADMIN')`

File nguồn tham chiếu:
- FE:
  - `frontend/src/pages/admin/AdminTicketsPage.tsx`
  - `frontend/src/pages/admin/AdminTicketDetailPage.tsx`
  - `frontend/src/services/legalTicket.service.ts`
  - `frontend/src/services/user.service.ts`
- BE:
  - `backend/src/main/java/com/analyzer/api/controller/admin/AdminTicketManagementController.java`

Điểm chưa rõ:
- `ĐÃ RÕ`: Lấy thông tin expert từ API `GET /api/v1/admin/users/experts` để có thông tin chuyên ngành.

Checklist FE:
- Thay thế API `getUsers()` thông thường bằng API `/admin/experts` khi mở panel assign ticket.
- Cập nhật hiển thị dòng option trong select box hoặc combobox chọn expert: Hiển thị thêm Chuyên ngành và Ngành luật.
- Nên disable action theo status ticket nếu backend có state machine chặt.

Tiêu chí hoàn thành:
- Admin xem được list và detail.
- Admin phân công được đúng Expert dựa vào thông tin chuyên ngành hiển thị trực quan.

HIỆN TRẠNG:

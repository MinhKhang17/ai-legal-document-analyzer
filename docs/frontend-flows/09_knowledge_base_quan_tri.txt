FLOW 09 - KNOWLEDGE BASE QUẢN TRỊ

Mục tiêu:
- Cho admin tạo knowledge entry, upload hoặc ingest nội dung, theo dõi version, review, publish, unpublish và archive.
- Cho FE hiển thị rõ lifecycle nội dung knowledge thay vì coi đây là một form upload đơn giản.

Actor:
- `ADMIN`

Điểm bắt đầu:
- Route `/knowledge-base`
- Route `/knowledge-base/:id`

Điều kiện tiên quyết:
- User có role admin.
- Với upload text, FE cần `user.id` để gắn `createdById`.
- Với ingest file, FE cần file đầu vào hợp lệ và AI ingestion endpoint khả dụng.

Luồng tổng quát:
1. `KnowledgeBasePage` tải danh sách knowledge entry theo trang.
2. Admin nhập form text-based gồm `code`, `title`, `category`, `scope`, `extractedContent`, `rawContent`.
3. FE gọi `uploadKnowledgeBaseEntry(payload)` để tạo version knowledge ban đầu.
4. Nếu admin chọn file, FE đọc file cục bộ, tự điền `title`, `code` và `rawContent`/`extractedContent` gợi ý trước khi upload.
5. Sau khi upload xong, nếu có file đính kèm thì FE tạo ingest job background bằng `ingestKnowledgeBaseEntry()` rồi đẩy file lên AI service bằng `ingestKnowledgeBaseFile()`.
6. `KnowledgeBasePage` lưu `activeJob` vào localStorage và poll tiến trình job cho đến khi `INGESTED` hoặc `FAILED`.
7. Admin mở detail entry để xem entry, version history và các tài liệu đã ingest.
8. `KnowledgeBaseDetailPage` cho phép review, publish, unpublish và archive theo trạng thái hiện tại.
9. FE dùng `canKnowledgeAction()` để quyết định action nào được render theo lifecycle status.
10. Khi thao tác thành công, FE reload detail và version list để đồng bộ lại trạng thái.

Sơ đồ text:
- `knowledge list -> upload entry -> optional file ingest -> detail -> review -> publish/unpublish/archive`

Màn hình / component:
- `frontend/src/pages/knowledge-base/KnowledgeBasePage.tsx`
- `frontend/src/pages/knowledge-base/KnowledgeBaseDetailPage.tsx`
- `frontend/src/components/knowledge-base/KnowledgeBaseIngestedDocumentsCard.tsx`

Dữ liệu / state:
- List page: `entries`, `totalItems`, `totalPages`, `form`, `selectedFile`, `activeJob`
- Detail page: `entry`, `versions`, `note`

Validation:
- Upload yêu cầu `user.id`, `code`, `title`, `extractedContent`.
- Ingest yêu cầu `requestId`.
- Publish và archive yêu cầu `note`/`reason` ở FE.
- `reviewKnowledgeBaseEntry()` hiện đang chạy theo nhánh approve mặc định.

API / service tích hợp:
- FE:
- `frontend/src/services/knowledgeBase.service.ts`
- BE:
- `GET /api/v1/admin/knowledge-base`
- `POST /api/v1/admin/knowledge-base/upload`
- `GET /api/v1/admin/knowledge-base/{id}`
- `GET /api/v1/admin/knowledge-base/{id}/versions`
- `POST /api/v1/admin/knowledge-base/{id}/ingest`
- `POST /api/v1/admin/knowledge-base/{id}/review`
- `POST /api/v1/admin/knowledge-base/{id}/publish`
- `POST /api/v1/admin/knowledge-base/{id}/unpublish`
- `POST /api/v1/admin/knowledge-base/{id}/archive`
- `GET /api/v1/admin/knowledge-base/ingestion-jobs/{jobId}`
- `POST /api/v1/admin/knowledge-base/ingestion-jobs/{jobId}/failed`
- `GET /api/v1/admin/knowledge-bases/{knowledgeBaseEntryId}/ingested-documents`

UI states:
- List table có pagination.
- Card upload entry với default state `PENDING / PRIVATE / INACTIVE`.
- Card progress cho background ingest job.
- Detail card + version table + latest extracted content.
- Card quản lý lifecycle gồm approve/review, publish, unpublish, archive.

Edge cases:
- Flow upload hiện thiên về text payload, nhưng FE còn hỗ trợ chọn file để đẩy qua AI ingestion service.
- File text được đọc bằng browser trước khi submit; file nhị phân sẽ chỉ có thông tin gợi ý để AI xử lý tiếp.
- Background job được lưu vào localStorage để refresh trang vẫn theo dõi tiếp được.
- Nếu job thất bại, FE đánh dấu job failed và vẫn reload entry list để đồng bộ.
- `KnowledgeBaseIngestedDocumentsCard` là phần riêng để xem tài liệu đã ingest, có filter theo keyword/status/visibility.

Business rules:
- Flow này là admin-only.
- Có lifecycle version/status/reviewDecision/publish/unpublish/archive.
- Không nên coi upload knowledge là một form text thuần tuý nếu business muốn hỗ trợ file import.
- Action render theo status, không cho admin bấm sai thứ tự.

Quyền:
- `@PreAuthorize("hasRole('ADMIN')")`

Điểm chưa rõ:
- `CHƯA XÁC ĐỊNH`: business muốn knowledge upload chính là nhập text tay, import file, hay cả hai như hiện tại FE đang hỗ trợ.
- `CHƯA XÁC ĐỊNH`: endpoint `ingestedDocuments` có cần tài liệu API đầy đủ hơn nếu FE tiếp tục mở rộng filter và mapping theo version hay không.

Checklist FE:
- Hiển thị rõ trạng thái job ingest background.
- Phân biệt rõ entry version, ingestion job và tài liệu đã ingest.
- Đừng hard-code action; hãy bám `canKnowledgeAction()` theo status backend.
- Khi update detail, phải reload cả versions lẫn card ingested documents.

Tiêu chí hoàn thành:
- Admin tạo entry, xem detail, ingest, review, publish, unpublish và archive được.

HIỆN TRẠNG:
- Có flow quản trị knowledge thật, và FE hiện đã hỗ trợ cả text upload lẫn background file ingestion.

SUY LUẬN:
- Đây là flow quản trị nội dung có lifecycle rõ, nhưng UI phải bám trạng thái backend rất chặt để tránh render sai action.

ĐỀ XUẤT:
- FE nên chốt lại với business xem UX ưu tiên text entry hay file import, vì hiện tại code đang đi theo hướng hỗn hợp.
